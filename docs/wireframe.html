System Design Draft

Tag: architecture | Created: Today, 14:30

The core philosophy here is offline-first. We need to ensure that the optimistic UI renders instantly without waiting for the Supabase network confirmation.

Local state management will be handled via standard React context, with a sync queue running in the background. If the user creates a note, it gets a temporary client ID. Once the server confirms, we map the client ID to the real UUID.

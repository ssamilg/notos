# ADR-007: Note Detail UX Optimization & Accessibility

**Date:** 2026-07-08
**Status:** Accepted

## Context
The Note Detail view requires layout adjustments to improve visual hierarchy, navigation, and accessibility without discarding the existing robust interaction state. The previous design utilized a large title and a distinct back button, consuming excessive vertical space. Furthermore, newly created empty notes presented a blank canvas failing to demonstrate Markdown capabilities, keyboard navigation lacked logical sequencing, and the "Mark Done" action lacked offline-first optimistic rendering.

## Decision
1. **Header Redesign (Breadcrumbs):** Deprecate the large Note Title and standalone Back button. Implement a monospace breadcrumb navigation path (`← [Project Name] / [Note Title]`) with prominent, readable tags positioned directly beneath it.
2. **Layout Shift & Styling:** Move the primary action buttons to a right-aligned vertical stack. Introduce subtle corner borders framing the main content area to match the new wireframe aesthetic.
3. **Logic Preservation:** Strictly retain the existing in-place editing functionality and the `isEditing` state toggle. The wireframe visually displays both `EDIT` and `SAVE` for layout purposes, but the application will continue to conditionally swap these buttons based on the active mode.
4. **Empty State Seeding:** Upon optimistic creation, seed the new note payload with a default Markdown template.
5. **Accessibility:** Enforce a strict tab-indexing flow for keyboard users: Breadcrumb Title -> Tags Input -> Content Area -> Action Menu.
6. **Bug Fix:** Refactor the "Mark Done" and "Save" mutation to utilize React Query's `onMutate` for instant, offline-first cache updates.

## Consequences

### Positive
- Breadcrumb navigation saves vertical space and reinforces a terminal-like aesthetic.
- Seeding new notes with default content serves as frictionless onboarding for Markdown capabilities.
- Strict tab indexing dramatically improves the rapid, keyboard-only capture workflow.
- Preserving existing state logic minimizes regression risks and speeds up implementation.

### Negative
- Managing explicit `tabIndex` requires careful overrides of default browser focus behavior.
